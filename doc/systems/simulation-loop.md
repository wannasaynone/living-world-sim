# Simulation Loop System

## 系統目的

Simulation Loop 負責推進世界時間，讓角色依照自身狀態、偏好、個性傾向、所在地點、可用活動、可用物件與其他角色狀況，自動選擇下一個行動，並將結果寫入事件日誌。

此系統服務 Game Core 的「自訂內容 -> 自由模擬 -> 觀察事件」循環。

## 核心循環

每個模擬 tick 依序執行：

1. 推進世界時間。
2. 更新角色狀態與情境修正。
3. 為每個可行動角色建立候選行動清單。
4. 根據角色個性、偏好、狀態、地點、關係與隨機權重選擇行動。
5. 套用行動結果。
6. 產生事件日誌。
7. 等待下一個 tick。

## Tick

Tick 是文字版第一版的最小模擬單位。

第一版不使用即時秒數作為設計核心，而是使用離散步進。玩家可以手動推進一步，也可以讓系統自動連續推進。

每個 tick 至少會產生：

- 世界時間變化。
- 零個或多個角色行動。
- 零個或多個角色互動。
- 對應事件日誌。

## 角色行動來源

角色的候選行動來自四種來源：

- 角色自身可執行的基本行動。
- 目前地點提供的活動。
- 目前地點內物件提供的行動。
- 其他角色觸發的互動。

第一版基本行動包含：

- 前往某個地點。
- 參加目前地點的活動。
- 使用目前地點的物件。
- 與同地點角色交談。
- 暫時無事可做。

## 個性傾向

角色個性使用 8 個可讀數值表示：

- I：內向
- E：外向
- S：實感
- N：直覺
- T：思考
- F：情感
- J：判斷
- P：感知

每個數值為 0 到 100。

系統顯示可以組合出玩家熟悉的 MBTI 標籤。例如 I 高於 E、N 高於 S、T 高於 F、J 高於 P 時，顯示為 INTJ。

但系統判斷不只看四字母標籤，而是使用 8 個實際數值。

同一組對立傾向建議維持可比較關係：

- I / E
- S / N
- T / F
- J / P

第一版不強制兩者總和等於 100，但 UI 應讓玩家看得出哪一側較強。

## 基礎個性與情境修正

角色有一組穩定的基礎個性值，代表長期傾向。

模擬過程中，事件可以產生暫時性的情境修正值，但不直接改寫基礎個性。

實際行動判斷使用：

```text
基礎個性值 + 情境修正值 + 當前狀態影響
```

例如：

```text
角色基礎值：
I 80 / E 30

情境修正：
剛和熟人聊得很開心，E +20，持續 3 tick

實際判斷：
I 80 / E 50
```

這讓角色能在特定情境下短暫改變行為傾向，但不會因短期事件永久改變人格。

第一版不處理永久人格成長。永久變化可以留給後續的成長、創傷、長期關係或劇情事件系統。

## 行動選擇

每個候選行動都會得到一個權重分數。

第一版權重來源包含：

- 角色個性傾向。
- 角色偏好。
- 角色目前狀態。
- 行動所在的地點。
- 行動涉及的物件。
- 行動涉及的話題。
- 行動涉及的其他角色。
- 隨機變動。

系統不保證角色永遠選擇最高分行動，而是依權重抽選，讓角色看起來有傾向但不完全機械。

個性傾向會影響候選行動權重，但不會直接禁止行動。例如 I 高的角色仍然可以主動交談，只是權重較低；E 高的角色也可以選擇獨處，只是權重較低。

## 狀態

角色至少需要以下狀態：

- 所在地點。
- 心情。
- 精力。
- 當前活動。
- 最近互動對象。
- 最近話題。
- 暫時情境修正。

第一版狀態只需足以影響文字事件，不需要完整生活模擬數值。

## 互動

角色互動發生於兩名或多名角色位於同一地點時。

第一版互動類型包含：

- 開啟話題。
- 回應話題。
- 一起參加活動。
- 一起使用物件。
- 因偏好、個性或狀態產生簡單反應。

互動結果必須寫入事件日誌，並可改變角色狀態、關係數值或暫時情境修正。

## 事件日誌

事件日誌是第一版最重要的輸出。

每筆事件至少包含：

- 發生時間。
- 事件類型。
- 參與角色。
- 發生地點。
- 涉及物件。
- 涉及活動。
- 涉及話題。
- 顯示文字。
- 結果摘要。

若某欄位不適用，可以留空。

## 視覺擴充預留

Simulation Loop 不直接依賴圖片或動畫，但事件結果需要預留視覺提示欄位。

每筆事件可以預留：

- `portraitCueId`
- `locationImageCueId`
- `objectImageCueId`
- `animationCueId`
- `soundCueId`

第一版可以不使用這些欄位，但資料格式應允許未來加上演出層。

## 第一版完成條件

本系統第一版完成時，必須能做到：

- 玩家建立的角色會在多個地點間移動。
- 角色會根據個性、狀態、地點活動與物件行動產生不同事件。
- 同地點角色有機會互動。
- 互動會使用玩家定義的話題。
- 互動可以造成暫時情境修正。
- 系統會持續產生可閱讀的事件日誌。
- 模擬可以手動推進，也可以自動連續推進。
