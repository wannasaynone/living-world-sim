# Game Core

## 核心體驗

因為玩家可以定義角色、話題、物件、地點與活動規則，並觀察角色依照這些設定自由行動與互動，產生「自己創造了一個會自己運轉的小型社群」的觀察、編劇與意外感。

## 設計目標

本專案是一款以文字為第一呈現形式的模擬遊戲。玩家不是直接操控單一角色，而是建立世界中的角色、地點、物件、話題與活動規則，然後觀察角色如何在這些條件下自行移動、選擇活動、使用物件、開啟話題並互相互動。

第一版重點不是畫面演出，而是讓模擬規則、資料結構與事件日誌成立。後續可以在不推翻核心資料模型的前提下，擴充角色圖片、地點背景、物件圖示、動畫提示與更豐富的視覺呈現。

## 玩家幻想

玩家像是在設計一個小型社交生態箱：

- 建立不同性格、偏好、關係與話題傾向的角色。
- 設定每個地點有哪些活動與物件。
- 定義物件可以觸發哪些行動。
- 觀察角色基於狀態、偏好、地點與關係做出行動。
- 從事件日誌中看到預期內與預期外的互動結果。

## 第一版範圍

第一版只要求文字版本可玩：

- 可以建立與編輯角色。
- 可以建立與編輯話題。
- 可以建立與編輯地點。
- 可以建立與編輯物件。
- 可以定義物件對應行動。
- 可以定義地點內含活動。
- 角色會自動選擇地點、活動、物件行動與互動對象。
- 系統會產生可閱讀的事件日誌。

第一版不要求：

- 角色立繪。
- 地點背景圖。
- 物件圖示。
- 動畫。
- 複雜 UI 演出。
- 即時 3D 或 2D 場景呈現。

但資料結構需要預留未來擴充圖片與動畫的欄位。

## 核心限制

所有系統都必須服務於「自訂內容 -> 自由模擬 -> 觀察事件」這條核心循環。

若某個功能不能強化以下任一項，暫不納入第一版：

- 玩家能更清楚地定義世界內容。
- 角色能更合理地自行行動。
- 角色互動能產生更有趣的事件。
- 玩家能更容易觀察、理解、回顧模擬結果。
